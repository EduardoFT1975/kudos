PROJECT_AUDIT.md
Auditoría Global del Proyecto TimeWisdom / Kudos
Fecha: 7 de mayo de 2026
Estado: Pre-lanzamiento - Presupuesto 0

1. ARQUITECTURA ACTUAL
- Framework: Django 4.2+ (monolítico)
- Aplicaciones: 1 app principal (kudos_app) con todo el código
- Modelos: 38
- Vistas: 99
- Rutas: 92
- Plantillas HTML: 114
- Base de datos: SQLite (4 migraciones)
- Servidor: Preparado para Gunicorn + WhiteNoise
- IA: Agentes y comandos manage.py (muchos en desarrollo)

2. FORTALEZAS
- Configuración dev/prod bien pensada (settings.py)
- Soporte PostgreSQL preparado
- Comandos manage.py potentes para importación
- Mapa y cápsulas parcialmente implementados
- Contenido inicial importado posible

3. DEUDA TÉCNICA
- Todo en una sola app → alto acoplamiento
- SECRET_KEY con fallback inseguro
- Sin tests automatizados visibles
- Dependencias IA y Celery comentadas
- Importaciones 24/7 no preparadas para producción

4. RIESGOS ACTUALES
- Seguridad (SECRET_KEY, permisos, social login)
- Escalabilidad (SQLite + bucles infinitos)
- Mantenibilidad (99 vistas en un archivo)
- Costes ocultos si se activa OpenAI

Diagnóstico general: Proyecto ambicioso con buena base pero necesita refactor urgente y reducción de scope controlada.
