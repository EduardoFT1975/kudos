Plan de Importación Masiva de Contenido (Fase 1)

Objetivo: Crear una base de datos amplia de cápsulas públicas antes del lanzamiento.

FASE 1A - Fuentes Prioritarias (Semanas 3-6)
1. Wikipedia (usando API)
   - Artículos geolocalizados
   - Categorías: Historia, Arte, Ciencia, Monumentos, etc.
2. Overpass API / OpenStreetMap (POI gratuitos)
   - Museos, monumentos, sitios históricos, puntos de interés
3. UNESCO World Heritage (lista oficial)
4. Imágenes: Wikimedia Commons (licencias libres)

FASE 1B - Enriquecimiento
- Para cada POI/artículo → crear Capsule
- Generar:
  - Resumen corto
  - Clip de voz (15s y 60s)
  - Imagen principal
  - Metadatos 6D (tiempo histórico aproximado, mérito básico, etc.)
- Guardar siempre fuente original + licencia

Estrategia Técnica:
- Usar comandos manage.py (ej: import_wikipedia, import_poi, enrich_capsules)
- Scheduler básico con APScheduler (sin Celery al inicio)
- Rate limiting fuerte para no ser bloqueados
- Procesar por lotes (batches) para no saturar el servidor

Prioridad de importación:
1. Europa + grandes ciudades (alta densidad de contenido)
2. Sitios UNESCO
3. Monumentos y museos mundiales
4. Eventos históricos importantes

Métrica de éxito Fase 1: +50.000 cápsulas públicas con clips generados antes del lanzamiento.

Nota importante: Empezar con volúmenes controlados (miles, no millones) para validar el pipeline.
